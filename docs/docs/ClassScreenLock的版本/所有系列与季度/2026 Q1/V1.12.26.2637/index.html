V1.12.26.2637

更新类型: 正式非稳定版本​

发布日期: 2026-03-07​

系列代号: V1系列 - Creeper​

[!CAUTION]

⚠️ 重要提示：非稳定版本​

本版本虽然标记为“正式版本”，但属于非稳定构建版 (Beta)。请在虚拟环境或非生产环境中先行测试。

请注意以下风险：

  • 新增的“看门狗”和“Web集控”功能涉及系统底层权限及网络通信
  • 部署前请确保防火墙已放行相应端口
  • 如遇进程崩溃、连接失败或数据异常，请立即停止使用并前往 Issues 页面 报告

🎯 更新摘要​

[!IMPORTANT]

本次更新是 ClassScreenLock 迈向“智能化管控”的关键一步​

🌐 全新 Web 集控管理平台​

发布配套网���项目，实现浏览器端的远程可视化管理

🛡️ 安全加固​

引入看门狗机制，防止恶意终止进程；增加数据完整性校验与备份

🔧 问题修复​

问题类型修复详情
🛡️ 进程守护失效修复了在部分 Windows 版本下，软件容易被任务管理器强制结束且无法自动重启的问题
💾 配置丢失修复了非正常关闭（如断电、强制杀进程）后，本地配置文件损坏导致设置重置的 Bug

✨ 功能更新​

🌐 Web 集控管理平台 (Centralized Control Web Portal)​

这是本次更新的重头戏！ 我们开发了一个独立的 Web 网站项目，作为所有客户端的指挥中枢。

📊 可视化大屏监控​

  • 实时状态: 管理员可在网页端实时查看所有在线客户端的状态（在线/离线/已退出）

🎮 远程指令下发​

  • 策略同步: 在网页端修改安全配置或网络拦截策略后，客户端将在每次重启时自动拉取同步，无需单机配置

🔐 设备接入认证​

  • 密钥配对: 客户端加入集控需输入 Web 平台生成的ID，确保只有授权设备能加入集控网络
  • 分组管理: 支持在网页端将设备划分为不同组（如“高一机房”、“图书馆终端”），实现分权管理

🛡️ 智能看门狗机制 (Watchdog)​

描述: 新增独立守护进程，实时监控主程序状态。一旦检测到主进程被恶意关闭或意外崩溃，将会自动拉起恢复。

💾 数据防篡改与备份系统​

描述: 引入关键数据的实时校验。若检测到文件被外部程序替换或删除，系统将自动从隐藏备份区恢复原始文件。

好处: 有效防止用户通过删除配置文件来绕过锁屏限制

🎨 界面与体验优化​

优化项详情
🌓 Web 端界面采用现代化的 Dashboard 设计风格，支持深色/浅色模式切换
🖥️ 客户端侧边栏增加了软件端集控管理的侧边栏，可以通过软件的侧边栏进入集控管理中心进行集控绑定等操作

🔍 已知问题​

[!important]

请在测试过程中留意以下问题​

问题说明
🌐 Web 平台部署目前 Web 项目需要手动部署在服务器（如 IIS/Nginx/Docker），尚未提供一键安装包
📡 网络依赖若客户端与 Web 服务器网络不通或无集控需求，可以选择不使用集控，不会影响客户端的单机功能
🔄 浏览器兼容Web 管理后台建议在Chromium 130+运行，不建议也不兼容IE
